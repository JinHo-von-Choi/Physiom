## 보안 취약점 제보

보안 취약점을 발견하신 경우, **공개 이슈로 제보하지 마시고** 비공개로 연락 부탁드립니다.

### 제보 방법
- 이메일: jinho.von.choi@nerdvana.kr
- 제목: `[SECURITY] 프로젝트명 - 취약점 요약`

### 제보 시 포함할 정보
1. 취약점 설명
2. 영향 범위 (심각도)
3. 재현 방법
4. 가능한 경우 수정 방법 제안

### 대응 절차
1. 24시간 내 제보 확인 회신
2. 7일 내 취약점 검증
3. 30일 내 패치 배포 (심각도에 따라 조정)
4. 패치 배포 후 공개 공지

## 보안 고려사항

### 프로젝트 보안 기능
- ✅ 원본 이미지 메모리 내 처리 (디스크 미저장)
- ✅ 임베딩 역추적 어려움 (고차원 벡터)
- ✅ 예외 스택 트레이스 최소 노출
- ✅ 스레드 안전 설계
- ❌ API 인증 미구현 (프로덕션 배포 시 추가 필요)
- ❌ HTTPS 미설정 (Nginx 리버스 프록시 권장)
- ❌ Rate limiting 미구현

### 프로덕션 배포 시 필수 보안 조치
1. **API 인증**: Bearer Token 또는 API Key 추가
2. **HTTPS**: TLS 1.2 이상 사용
3. **Rate Limiting**: slowapi 등으로 요청 제한
4. **입력 검증**: 경로 순회 공격 방지
5. **로그 보안**: 민감 정보 마스킹
6. **DB 암호화**: 임베딩 저장 시 AES-256 암호화

## 알려진 제한사항
- InsightFace 모델 파일은 공개 저장소에서 다운로드 (무결성 검증 권장)
- 임베딩은 생체정보로 GDPR/개인정보보호법 적용 대상

## 감사
보안 취약점을 책임있게 제보해주시는 분들께 감사드립니다.
